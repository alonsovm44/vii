IO Language Specification v1.0.0
Classification: Concatenative, Indentation-based, Turing-Complete, General-PurposeDesign Goal: Absolute concept minimalism. Zero syntactic boilerplate.

1. The Philosophy
IO measures minimalism by concept count, not character count. It strips away OS-level engineering artifacts (types, brackets, punctuation, module systems) to expose pure, readable logic. If a concept does not strictly serve conditional branching, memory allocation, or basic I/O, it is excluded.

2. The 3 Inviolable Rules
No Punctuation: There are no { }, [ ], ( ), ;, ,, or :. Punctuation is the primary cause of beginner syntax errors.
Implicit Output: If a line evaluates to a value and is not assigned to a variable, it is automatically printed to STDOUT. There is no print() function.
Indentation Only: Spaces (recommended: 2) are the only structural syntax used to define blocks of code.
3. The Complete Vocabulary (20 Words)
IO possesses exactly 20 reserved words/symbols. There are no keywords for class, return, import, true, false, try, or variable types. (Note: 0 is false. Any non-zero number is true. These 20 words cannot be used as variable names).

3.1 Data
Numbers: Integers (e.g., 5, -42, 1024)
Text: Strings wrapped in double quotes (e.g., "hello world")
3.2 Assignment & Logic
= : Assigns the right side to the left side. Suppresses implicit output.
== : Equality check. Yields 1 for true, 0 for false.
< : Less than. Yields 1 for true, 0 for false.
> : Greater than. Yields 1 for true, 0 for false.
3.3 Math
+ : Addition (Also functions as string concatenation for text).
- : Subtraction.
* : Multiplication.
/ : Integer Division.
% : Modulo (Remainder).
3.4 Control Flow
if : Begins a conditional block. Requires a preceding logic evaluation.
else : Begins the fallback conditional block.
while : Begins an infinite repetition block. Requires a preceding logic evaluation. Loop terminates when the evaluation yields 0.
3.5 Abstraction
do : Defines a reusable function. Arguments are space-separated. The final evaluated line inside the do block is implicitly returned. There is no return keyword.
Syntax: do function_name arg1 arg2
3.6 Memory & Data Structures
list : Initializes an empty list in memory.
at : Reads a value from a list or string by index.
Syntax: list at index
set : Writes a value to a list by index.
Syntax: list set index value
3.7 Universal I/O (The ask Polymorphism)
ask : Pulls data into the program. Behavior depends on arguments:
x = ask : Pauses execution and waits for keyboard input.
x = ask "path.txt" : Reads the entire contents of a disk file into x.
3.8 Disk Output
put : Pushes data out of the program to a disk file. Overwrites existing files. Suppresses implicit output.
Syntax: put "path.txt" "data to write"
3.9 Environment
arg : A built-in list containing Command Line Interface arguments passed to the program.
paste : Compiler directive. Injects the source code of another file directly into the current file at compile time. No namespaces.
Syntax: paste "helper.io"
3.10 Meta
# : Single-line comment. The compiler ignores everything from # to the end of the line.
4. Formal Syntax Examples
4.1 Hello World
"Hello World"
(Evaluates to a string. Rule #2 prints it.)

4.2 Variables & Math
```io
x = 5 + 5
x * 2
```

5. Compiler Constraints
An IO v1.0.0 compliant compiler must:

Reject any syntax containing punctuation {,},[,],(,),;,:.
Evaluate expressions strictly left-to-right without operator precedence (e.g., 2 + 3 * 4 evaluates to 20, not 14). Parentheses are not allowed to force precedence; use variables instead.
Treat the ask keyword polymorphically based on the presence of an adjacent string literal.
Execute paste at compile-time (text substitution), not at runtime.
