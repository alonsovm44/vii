# Vii v1.2 — "Code Anything" Roadmap

> **Philosophy guardrail**: Every new word must unlock an entire category of programs that were previously impossible or impractical in vii. If a feature can be built as a `.vii` library, it should be. New keywords are the last resort.

---

## Gap Analysis: What You Can't Code Today

| Category | Current State | What's Blocked |
|----------|--------------|----------------|
| Key-value data | Only lists (numeric index) | JSON, databases, caches, configs, graphs, frequency counts, lookup tables |
| Loop control | `while` only, no early exit | Search algorithms, parsers, state machines — all need `break` |
| Iteration | Manual `i = 0; while i < len lst` | Readability, beginner ergonomics, off-by-one errors |
| Negative numbers | `0 - x` workaround | Math expressions, physics, coordinates, temperatures |
| Substrings | Single char via `at` only | Parsers, tokenizers, text processing |
| Type introspection | None | Polymorphic functions, safe input handling |
| File append | Read → concat → write | Log files, data pipelines |
| Time | None | Benchmarks, timestamps, scheduling, games |
| Error recovery | `exit(1)` on any error | Long-running servers, robust CLI tools |
| String comparison | `==` and `!=` only | Sorting, natural ordering, binary search on strings |
| Self-hosting | Prerequisites done, not started | Bootstrapping, dogfooding |

## New Keywords (3 words)

These are the only additions to the reserved word list. Each unlocks a fundamentally new capability.

### 1. `dict` — Key-Value Maps

**Why**: Dictionaries are the #1 missing data structure. Without them, JSON parsing, frequency counting, memoization, config handling, and graph algorithms are all impractical. Lists with numeric indices can't replace string-keyed lookups.

**Syntax**:
```vii
# Create empty dict
d = dict

# Set key-value pair
d key "name" "Vii"
d key "version" 1.2

# Get value by key
d at "name"          # "Vii"

# Dict with initial data (list of [key, val] pairs)
d = dict
pairs = list
pair = list
pair set 0 "color"
pair set 1 "blue"
pairs set 0 pair
d from pairs
```

**Implementation**: `dict` creates a `VAL_DICT` value kind. Internally uses the existing `Table` hash map (already in `interp.c`). `key` is a postfix operator like `set` — `expr key key_expr val_expr`. `at` already works on dicts (extend `ND_AT` to check dict kind). `from` populates dict from a list of 2-element lists.

**AST nodes**: `ND_DICT`, `ND_KEY`  
**Value**: Add `VAL_DICT` to `ValKind`, add `Table *dict` field to `Value` struct.

**Keyword count**: +2 (`dict`, `key`)

### 2. `break` — Early Loop Exit

**Why**: Without `break`, every loop that might exit early requires a `done = 0` flag variable and `while condition and done == 0`. This is the #1 friction point in real programs. Search algorithms, parsers, input validation — all need early exit.

**Syntax**:
```vii
# Find first even number
i = 0
while i < len nums
  if nums at i % 2 == 0
    result = nums at i
    break
  i = i + 1
```

**Implementation**: `break` sets a flag in the interpreter loop. In `eval` for `ND_WHILE`, check the flag after each body iteration. If set, clear it and return. Add `TOK_BREAK`, `ND_BREAK`.

**Keyword count**: +1 (`break`)

## New Built-in Operations (0 new keywords)

These extend existing keywords or add built-in functions (callable like `len`, `ord`, etc.) without adding reserved words.

### 3. `slice` — Substring / Sublist

**Why**: Extracting parts of strings and lists is fundamental. Currently impossible for strings (only single-char `at`), and requires manual loops for lists.

**Syntax**:
```vii
s = "Hello World"
slice s 0 5          # "Hello"
slice s 6 11         # "World"

nums = list
nums set 0 10
nums set 1 20
nums set 2 30
slice nums 1 2       # [20]
```

**Implementation**: Add `TOK_SLICE`, `ND_SLICE`. Like `len`, it's a prefix built-in. `slice expr start end` returns a new string or list from index `start` (inclusive) to `end` (exclusive).

**Keyword count**: +1 (`slice`)

### 4. `type` — Type Introspection

**Why**: Without knowing what type a value is, you can't write polymorphic functions or safely handle user input that might be a number or string.

**Syntax**:
```vii
type 42              # "num"
type "hello"         # "str"
type list            # "list"
type dict            # "dict"
```

**Implementation**: Add `TOK_TYPE`, `ND_TYPE`. Returns a string: `"num"`, `"str"`, `"list"`, `"dict"`, `"none"`.

**Keyword count**: +1 (`type`)

### 5. `time` — Current Timestamp

**Why**: Without any time access, you can't benchmark code, generate unique seeds, timestamp logs, or build games with timing. This is a single built-in that unlocks an entire category.

**Syntax**:
```vii
t = time             # seconds since epoch (e.g., 1745089200)
```

**Implementation**: Add `TOK_TIME`, `ND_TIME`. Calls `(double)time(NULL)` in C. No arguments.

**Keyword count**: +1 (`time`)

### 6. `append` mode for `put`

**Why**: Currently `put` only overwrites files. Appending requires reading the entire file, concatenating, and writing back — impractical for logs and data pipelines.

**Syntax**: Extend `put` with an optional flag:
```vii
"path.txt" put "line 1"          # overwrite (existing behavior)
"path.txt" put "line 2" append   # append mode
```

**Implementation**: Add `TOK_APPEND`. When `put` sees `append` after the data expression, open with `"a"` mode instead of `"w"`.

**Keyword count**: +1 (`append`)

### 7. String comparison for `<`, `>`, `<=`, `>=`

**Why**: Sorting strings, comparing filenames, binary search on sorted data — all require ordered comparison. Currently these operators only work on numbers.

**Implementation**: In `eval` for `ND_BINOP`, when both operands are `VAL_STR`, use `strcmp` for comparison instead of numeric comparison. Zero new keywords.

**Keyword count**: +0

### 8. Unary minus

**Why**: `0 - x` is the #1 cited friction in the README tips. It's not about saving characters — it's about mathematical expressions reading naturally.

**Syntax**:
```vii
x = -5
y = -x
z = -(a + b)
```

**Implementation**: In the lexer, when `-` appears at the start of an expression (after `=`, operator, `(`, or at line start), emit `TOK_UMINUS`. In the parser, treat it as a prefix operator creating `ND_UMINUS` node. In `eval`, `ND_UMINUS` returns `val_num(-eval(n->left)->num)`. Alternatively, simpler: in `parse_primary`, if we see `TOK_MINUS`, consume it and negate the next primary.

**Keyword count**: +0

## Standard Library Expansions (0 new keywords)

All implemented as `.vii` files using existing + new built-ins.

### lib/dict.vii — Dictionary Utilities
```vii
paste "lib/dict.vii"

# has_key d k      — check if key exists
# keys d           — list of all keys
# vals d           — list of all values
# merge d1 d2      — merge two dicts (d2 wins on conflict)
# dict_from_list lst — list of [k,v] pairs → dict
# dict_to_list d   — dict → list of [k,v] pairs
```

### lib/json.vii — JSON Parser/Emitter
```vii
paste "lib/json.vii"

# json_parse s      — parse JSON string → dict/list/num/str
# json_stringify v  — convert dict/list/num/str → JSON string
```

This is the single most impactful library. JSON is the lingua franca of the web. With `dict` + `json.vii`, vii can talk to any API.

### lib/time.vii — Time Utilities
```vii
paste "lib/time.vii"

# now              — current epoch seconds (wraps built-in `time`)
# format_time epoch — epoch → "YYYY-MM-DD HH:MM:SS"
# parse_time s     — "YYYY-MM-DD" → epoch
# elapsed start end — difference in seconds
```

### lib/sort.vii — Sorting Algorithms
```vii
paste "lib/sort.vii"

# sort lst          — ascending sort (returns new list)
# sort_desc lst     — descending sort
# sort_by lst fn    — sort by key function
```

### lib/fs.vii — Filesystem Utilities
```vii
paste "lib/fs.vii"

# ls path           — list files in directory (via sys)
# mkdir path        — create directory
# rm path           — remove file
# cp src dst        — copy file
# mv src dst        — move/rename file
```

### lib/http.vii — HTTP Client (via curl + sys)
```vii
paste "lib/http.vii"

# http_get url      — GET request, returns body string
# http_post url body — POST request, returns body string
```

Built on `sys "curl -s ..."` — no native socket code needed.

### lib/test.vii — Testing Framework
```vii
paste "lib/test.vii"

# assert condition message — assert with message
# test name fn            — run a test function
# run_tests               — run all registered tests, report results
```

## What v1.2 Does NOT Add (Scope Creep Kill List)

These are deliberately excluded to preserve vii's minimalist identity:

- **Classes/OOP** — Use dicts + functions. Vii doesn't need `class`.
- **Exception handling** — Use `type` checks and `if` guards. `try/catch` adds 2 keywords for marginal gain.
- **Concurrency/threads** — Use `sys` to spawn processes. Threading is an OS concern.
- **First-class functions** — Functions are already callable by name. Passing them as values would require a `VAL_FUNC` type and breaks the "no punctuation" rule (callbacks need parens).
- **Pattern matching** — Use `if`/`else if` chains with `at` and `type`.
- **Modules/namespaces** — `paste` is sufficient. Taco handles distribution.
- **Operator precedence** — Left-to-right is simpler. Use intermediate variables.
- **Closures** — Functions already access parent scope via `Table.parent`.
- **Regex** — Build it in `lib/regex.vii` using `ord`/`chr`/`at` if needed. Not a language feature.
- **Float formatting** — Build `lib/fmt.vii` if needed. Not a language feature.
