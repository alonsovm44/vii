# Vii 1.4 TODO List

## Vision
Python syntax, C semantics. One language for beginners and experts.
Beginners write simple code, experts add annotations for control.

---

## Phase 0: Bundling ✅ (Primary Distribution Method)
- [x] Remove C codegen - bundling is simpler and more reliable
- [x] To make standalone executables, bundle the code + interpreter on a binary file
- [x] Add `vii script.vii --bundle [app | app.exe]` to make final binary
- [x] Self-extracting bundle detection at startup
- [x] Bundle format: [interpreter] + [script] + [footer: offset + magic "VIIBUNDL"]
- [x] Single source of truth - one interpreter, no C/Vii drift

## Phase 2: Memory Model (Critical)

### Stack Allocation ✅
- [x] Add `stack_alloc` builtin
- [x] Implement fixed arrays: `name type[N] = allocator` syntax (consistent with `x i32 = 5`)
  - Example: `buf u8[1024] = stack_alloc` - 1KB stack buffer
  - Example: `data f64[256] = stack_alloc` - 256 doubles on stack
- [x] Add array bounds checking (debug mode)
- [x] Generate alloca or VLAs in C

[DONE] on 26 of April 2026

### Heap Allocation
- [x] Add `heap_alloc` builtinl
- [x] Add `heap_free` builtin
- [x] Implement pointer type: `p ptr`
  Example use
  ```vii
  y i32 = 128
  x ptr i32 = addr y
  valof x # 128
  x # prints the hex adress of y
  ```
- [x] Add dereference operator: `valof(p)` or `valof p`. C equivalent = `*ptr`
- [x] Add address-of operator: `addr(x)` or `addr x`. C equivalent = `&x`
- [x] ptr + type return values for functions `do func->(ptr <type>) param`
- [ ] ptr + type parameter values for function arguments `do fun x->ptr i32 y->ptr f32
